# 인터페이스 보드 입력 라인 풀업/풀다운 조정 정리

작성일: 2026-05-13
대상 보드: 디지털 인터페이스 보드 (회로도: `20260429_회로도.pdf`)
대상 시스템: Jetson Orin Nano (Super) — `p3768-0000+p3767-0005-super`

---

## 0. 개요

본 문서는 인터페이스 보드의 **두 가지 입력 라인 조정 작업**을 함께 정리한 통합 보고서다.

| # | 대상 | 작업 | 방향 | 비고 |
|---|---|---|---|---|
| 1 | BUTTON1 / BUTTON2 (Pin 12 / 13, BCM18 / 27) | 외부 풀업 R 값 **감소** (10kΩ → 1kΩ) | 풀업 강화 | 별도 상세 보고서: `REPORT_button_pullup_change_20260430.md` |
| 2 | PIR (Pin 18, BCM24) | 신호선에 **외부 풀다운 10kΩ 신규 추가** | 풀다운 신설 | 본 문서 §2 |

두 작업의 **공통 배경**: Jetson Orin Nano 캐리어 핀에는 부팅 strap/ESD 목적의 잔여 풀 경로가 있어, 외부 보드의 풀이 약하면 분압이 발생해 SoC 입력이 의도된 레벨로 인식되지 않는다. 두 사례 모두 인터페이스 보드 자체는 무결이고, **외부 풀의 세기/방향 보완**으로 해결한다.

---

## 1. BUTTON1 / BUTTON2 — R17 / R27: 10kΩ → 1kΩ (풀업 강화)

### 1-1. 증상
- 결합 상태에서 버튼 미누름 시 idle V ≈ **1.0V** (기대 3.3V).
- 누름 후 뗌(릴리즈)했을 때 HIGH로 복귀하지 못함.

### 1-2. 원인
- **인터페이스 보드 자체는 무결** (단독 시험에서 0V/3.3V 깨끗).
- **Jetson 캐리어(P3768) 측 GPIO18/GPIO27 헤더 핀에 약 4.3kΩ 풀다운 path**가 존재.
- 외부 풀업 10kΩ과의 분압:
  ```
  V_idle = 3.3 × 4.3 / (10 + 4.3) = 0.99 V  → V_IH(≈2.0V) 미달 → LOW로 오인식
  ```
- SoC 내부 풀업(`nvidia,pull = <2>`)으로도 외부 4.3kΩ을 이기지 못함 (SoC pull-up 강도 ≈ 50~100kΩ 추정).

### 1-3. 처방
**R17, R27 두 부품을 각 10kΩ → 1kΩ으로 교체**.

| 항목 | 10kΩ | 1kΩ |
|---|---|---|
| idle V (결합 시) | 0.99V ❌ | **2.68V** ✓ |
| 누름 시 V | 0V | 0V |
| 누름 시 전류 (채널당) | 0.33mA | **3.3mA** |
| RC 시정수 (× C15 0.1µF) | 1ms | 0.1ms |

### 1-4. 적용 위치
- **R17**: BUTTON1 풀업 (Pin 12 / BCM18 / PH.07 라인)
- **R27**: BUTTON2 풀업 (Pin 13 / BCM27 / PY.00 라인)
- 두 위치 모두 회로 토폴로지 유지, 부품 값만 변경.

### 1-5. SW 측 함께 적용된 변경
DTBO `digital-interface-pinmux.dts`의 `button1_ph7`, `button2_py0` 노드에 `nvidia,pull = <2>` (SoC internal pull-up enable) 추가 적용. 외부 1kΩ과 병행하여 마진 확보.

### 1-6. 상세 근거
별도 보고서 `REPORT_button_pullup_change_20260430.md` 참조 (단계별 진단·대안 비교 포함).

## 2. PIR — 신호선에 외부 풀다운 10kΩ 신규 추가

### 2-1. 사전 작업 (HW 설계자 측 수정)
PIR 입력단의 BSS138 양방향 레벨시프터를 제거하고 직결로 변경:
- **R20 (10kΩ, +3.3V → GPIO24 풀업)** 제거
- **R26 (10kΩ, +5V → HW-MS1 풀업)** 제거
- **Q1 (BSS138 N-ch MOSFET)** 제거
- BSS138 풋프린트에서 소스–드레인 점퍼로 신호 직결 (PIR_OUT → GPIO24)

이 수정은 전기적으로 타당하다 — HW-MS03 모듈은 VCC=5V로 구동되지만 신호 출력이 **3.3V CMOS push-pull** (내부 LDO로 출력단 고정)이라 Jetson 3.3V GPIO와 직결 가능, 레벨시프터 불필요.

### 2-2. 수정 후 증상
- motion (감지): 신호 = **3.3V** (정상 HIGH)
- idle (미감지): 신호 = **2.4V** (기대 0V) ❌
- Jetson 입장에선 idle 2.4V > V_IH(≈2.0V) → **항상 HIGH로 오인식**, 모션 검출 불가.

### 2-3. 원인 — 두 약한 경로의 분압
DMM 진단 결과:

| 측정 | 결과 |
|---|---|
| (A) PIR 모듈 단독 출력 (보드 분리) | idle 0V / motion 3.3V (깨끗) |
| (B) 보드 PIR 신호 네트 ↔ +3.3V 저항 (Jetson 결합 상태) | **171kΩ** |
| (C) 보드 PIR 신호 네트 ↔ +3.3V 저항 (Jetson 분리 상태) | **∞ (OL)** |
| (D) 보드 단독, PIR 미연결, 전원 ON, 신호 네트 ↔ GND | 0V |

→ 171kΩ은 **Jetson PY.03 패드 고유 입력 누설** (`nvidia,pull = 0`이어도 잔존). 보드 자체엔 잔존 풀업 없음.

PIR의 idle LOW 구동력은 push-pull로 표시되지만 실측 등가 임피던스가 **~456kΩ급 매우 약함** (DMM 단독 측정 시엔 부하 없어 0V로 보였음).

```
V_idle = 3.3V × 456kΩ / (456kΩ + 171kΩ) ≈ 2.4V   ← 관측치와 일치
```

### 2-4. 왜 원래 BSS138 회로에선 잘 동작했나
BSS138 **바디 다이오드(소스=LV anode, 드레인=HV cathode)의 클램프 효과** 때문:
- PIR가 HV측을 LOW로 끌면 → 다이오드 순방향 도통 → R20(10kΩ) → +3.3V → 다이오드 → PIR LOW 핀으로 전류 흐름.
- LV측 GPIO24 = PIR_LOW + V_f ≈ **0.6V** → V_IL(≈1.0V) 안쪽 LOW.

즉 원래 회로의 LOW 인식은 R20/R26 풀업의 역할이 아니라 **BSS138 바디 다이오드의 능동 클램프**가 만들고 있었음. BSS138 제거로 이 메커니즘이 사라져, PIR의 약한 자체 LOW 구동만 남고 idle이 2.4V로 떠버린 것.

### 2-5. 처방
**PIR 신호선 ↔ GND 사이 10kΩ 외부 풀다운 1개 추가**.

검산:

| 상황 | 계산 | 결과 |
|---|---|---|
| idle | PIR(456k) ∥ 풀다운(10k) ≈ 9.78kΩ. 9.78kΩ ↔ 171kΩ to 3.3V 분압 | **0.18V** ✓ LOW |
| motion | PIR HIGH push-pull (low impedance) vs 10kΩ to GND. 3.3 × 10k/(10k+R_pir_hi) | **≈3.27V** ✓ HIGH |
| 풀다운 통과 전류 (motion) | 3.3V / 10kΩ | 0.33mA (무시 가능) |

### 2-6. 부착 위치 (정확한 가이드)

회로도 기준 PIR 부의 노드 구성:
```
R20 (제거): +3.3V (위 패드) ── (제거) ── GPIO24 노드 (아래 패드)
R26 (제거): +5V   (위 패드) ── (제거) ── HW-MS1 노드 (아래 패드)
Q1  (제거+점퍼): 소스(=GPIO24 노드) ── 점퍼 ── 드레인(=HW-MS1 노드)
SMW250-03 커넥터: [GND / signal / +5V]   ← 회로도 핀 순서
```

R20·R26 풋프린트는 한쪽 패드가 전원 레일이라 그대로 풀다운으로 재사용 불가 (전원 트레이스 절단 위험). 따라서 **신호 패드 한쪽만 활용 + GND로 별도 점퍼**.

**권장 부착 (R20 활용)**:
- **다리 1**: R20의 **아래쪽 패드** (= GPIO24 신호 네트)
- **다리 2**: SMW250-03의 **GND 핀 패드**
- 사이에 10kΩ을 디스크리트(SMD 또는 axial)로 납땜.

**R26 활용도 가능**하나, R20이 Jetson GPIO 입력단에 더 가까워 풀다운을 GPIO 직전에 거는 형태가 되어 노이즈 마진 측면에서 미세하게 유리.

### 2-7. 부착 전 식별 절차 (DMM)

보드 전원 OFF 상태:
1. R20의 두 패드 각각 ↔ SMW250-03 가운데 핀(=signal) 사이 도통 확인. **도통 있는 패드 = 신호 네트 쪽**. 반대쪽은 +3.3V.
2. SMW250-03의 3개 핀 중 어느 게 GND인지 확인: 보드의 큰 GND 영역(예: 12V GND 패드, RS232 GND)과 도통되는 핀 = GND.
3. R20 신호 패드 ↔ SMW250 GND 핀 사이 위치 확인, 10kΩ 부착.

### 2-8. 부착 후 검증

```bash
# 1) 정적 확인 (보드 전원 ON, PIR 연결, idle)
gpioget gpiochip0 125
# 기대: 0

# 2) 동적 확인 (모션 트리거)
gpiomon -b -F "%e" gpiochip0 125
# 손 흔들면: 1 (rising) → 수 초 후 0 (falling) 깔끔히 잡혀야 정상

# 3) DMM 점검 (납땜 양호도)
# 전원 OFF: 신호 네트 ↔ GND 저항 = 10kΩ ± 1%
```

### 2-9. SW 측 보조 (선택)

`02_interface/dtbo/digital-interface-pinmux.dts`의 `pir_py3` 노드에 `nvidia,pull = <1>` (pull-down) 추가 검토 가능. 단:
- SoC 내부 풀다운 강도는 수십 kΩ 수준 → 외부 10kΩ만큼 강하지 않음.
- 외부 풀다운만으로 충분히 동작하면 DT 변경 불요.
- 외부 풀다운 부착 전 임시로 적용해 동작 검증해 보는 용도로는 유용.

## 3. 공통 원리 — 외부 풀이 약하면 분압이 발생한다

두 작업의 본질은 동일하다: **Jetson 캐리어 핀 측에는 무시할 수 없는 잔여 풀 경로가 있고**, 외부 보드의 풀이 이를 압도할 만큼 강하지 않으면 라인 전압이 분압에 끌려 잘못된 레벨이 된다.

| 사례 | 외부 풀 | 캐리어 잔여 풀 | 결과 |
|---|---|---|---|
| BUTTON | 10kΩ pull-up to 3.3V | ~4.3kΩ pull-down | idle 1.0V (LOW로 처짐) |
| PIR (수정 후) | 없음 (제거됨) | ~171kΩ pull-up | idle 2.4V (HIGH로 떠버림) |

처방의 방향만 다를 뿐 원리는 같다:
- **BUTTON**: 풀업을 **더 강하게** (10kΩ → 1kΩ) → idle V가 3.3V 쪽으로 안전하게 끌림.
- **PIR**: 풀다운을 **새로 추가** (10kΩ) → idle V가 0V 쪽으로 안전하게 끌림.

두 경우 모두 SoC internal pull 설정만으로는 부족하여 **외부 R 부착이 필수**다.

## 4. 작업 진행 상태

| 작업 | 상태 |
|---|---|
| BUTTON R17/R27 → 1kΩ 교체 | HW 작업자 측 대기 (PCB 부품 교체 필요) |
| BUTTON DT pull-up (`nvidia,pull=2`) | DTBO 적용 완료 |
| PIR BSS138 + R20/R26 제거 및 점퍼 | HW 측 완료 |
| **PIR 풀다운 10kΩ 부착** | **HW 작업 대기 (본 문서 §2-6 위치 적용)** |
| PIR DT 설정 검토 (`nvidia,pull=1`) | 외부 풀다운 부착 후 동작에 따라 결정 |
