# 인터페이스 보드 변경요청서 - Jetson 전원 출력 변경 및 버튼 풀업 저항 추가

> 문서번호: CR-2026-001
> 작성일: 2026-03-07
> 요청자: 백산디지털
> 대상: 인터페이스 보드 (40핀 연결 커스텀 보드)
> 변경 항목: (1) Jetson 전원 출력 12V/3A 변경, (2) 버튼 GPIO 풀업 저항 추가, (3) PIR 센서 전원 3.3V→5V 변경, (4) RS232C 레벨 컨버터 수리, (5) 팬 구동 트랜지스터 교체

---

## 1. 변경 배경

### 1.1 기존 설계

인터페이스 보드는 **Jetson Nano** 기준으로 설계되었으며, 보드 내 DC-DC 컨버터를 통해 12V → 5V 변환 후 40핀 헤더 Pin 2, 4를 통해 Jetson에 5V 전원을 공급하는 구조이다.

| 항목 | 기존 설계 값 |
|------|------------|
| Jetson 모델 | Jetson Nano |
| Jetson 요구 전압 | 5V |
| Jetson 요구 전류 | 4A (최대 10W) |
| 인터페이스 보드 출력 | 5V / 7A (+5V/7A 레일) |
| 전원 공급 경로 | 40핀 헤더 Pin 2, 4 |

### 1.2 변경 사유

Jetson 보드가 **Jetson Orin Nano Super Developer Kit**으로 변경되었다. Orin Nano는 기존 Jetson Nano와 전원 사양이 상이하여 현재 인터페이스 보드의 5V 출력으로는 전원 공급이 불가능하다.

| 항목 | Jetson Nano (기존) | Jetson Orin Nano Super (변경) |
|------|-------------------|------------------------------|
| DC 입력 전압 | 5V | **9V ~ 20V** |
| 최대 소비 전력 | 10W | **25W** |
| 전원 입력 방식 | Micro-USB 또는 배럴잭 (5V) | **USB-C PD 또는 DC 잭 (9V~20V)** |
| 40핀 5V 핀 전류 | 출력 전용 | 입출력 가능, **최대 1A 제한** |

**핵심 문제:** Orin Nano는 최소 9V 이상이 필요하며, 40핀 헤더 5V 핀은 1A(5W)로 제한되어 25W 구동에 전력이 절대적으로 부족하다.

## 2. 변경 요청 사항

### 2.1 전원 출력 변경

기존 5V/7A 출력을 **12V/3A** 출력으로 변경하여 Jetson Orin Nano DC 잭에 직접 공급한다.

| 항목 | 기존 | 변경 요청 |
|------|------|----------|
| 출력 전압 | 5V | **12V** |
| 출력 전류 | 7A | **3A (최소), 권장 5A** |
| 출력 전력 | 35W | **36W (최소), 권장 60W** |
| 출력 커넥터 | 40핀 헤더 경유 | **DC 잭 케이블 (배럴 5.5x2.5mm 또는 USB-C)** |
| 공급 대상 | Jetson Nano (5V) | **Jetson Orin Nano Super (12V)** |

### 2.2 전류 산출 근거

```
Jetson Orin Nano Super 최대 소비 전력: 25W
12V 기준 전류: 25W / 12V = 2.08A
안전 여유 (20%): 2.08A x 1.2 = 2.5A
권장 설계 전류: 3A (최소), 5A (여유 확보)
```

### 2.3 전원 구조 변경 요약

```
[기존]
12V SMPS → J3 VIN → CONVERTER → 5V/7A → 40핀 Pin 2,4 → Jetson Nano

[변경 요청]
12V SMPS → J3 VIN ─┬→ LMS116MHX → 3.3V (인터페이스 보드 로직)
                    ├→ 5V 변환 → H-Bridge, 팬 등 (기존 유지)
                    └→ 12V/3A 출력 → Jetson Orin Nano DC 잭 (신규)
```

## 6. 버튼 GPIO 풀업 저항 추가

### 6.1 현재 문제

회로도 확인 결과, BUTTON1(GPIO18)과 BUTTON2(GPIO27) 라인에 **풀업 저항이 없다.**
기존 Jetson Nano에서는 소프트웨어 내부 풀업(`GPIO.PUD_UP`)으로 동작했을 수 있으나,
Jetson Orin Nano에서는 **내부 풀업이 기본 비활성**이며, `Jetson.GPIO` 라이브러리의
`PUD_UP` 파라미터가 **Orin 계열에서 동작하지 않는 것**으로 확인되었다.

```
현재 상태 (풀업 없음):
  3.3V ──── (연결 없음)
  GPIO18 ── BUTTON1 ── GND     버튼 미누름 시: GPIO 플로팅(불안정) ⚠️
                                버튼 누름 시: GPIO = LOW ✅
```

풀업 저항이 없으면 버튼을 누르지 않은 상태에서 GPIO 핀이 플로팅(고임피던스)되어
노이즈에 의한 오동작(허위 press/release 이벤트)이 발생한다.

### 6.2 변경 요청

BUTTON1, BUTTON2 각 GPIO 라인에 **3.3V 풀업 저항**을 추가한다.

| 항목 | 사양 |
|------|------|
| 풀업 전압 | 3.3V (인터페이스 보드 내부 3.3V 레일) |
| 풀업 저항 값 | **100kΩ** (권장) |
| 허용 범위 | 50kΩ ~ 100kΩ |
| 적용 대상 | BUTTON1 (GPIO18, Pin 12), BUTTON2 (GPIO27, Pin 13) |

> Jetson Orin Nano 데이터시트: "Any pull-up or pull-down resistors on the signals
> (except I2C) must be weak (limited to >50 kΩ)."
> 따라서 100kΩ을 권장한다.

### 6.3 변경 후 회로

```
변경 후:
  3.3V ──┬── R (100kΩ) ──┬── GPIO18 ── BUTTON1 ── GND
         │               │
         └── R (100kΩ) ──┴── GPIO27 ── BUTTON2 ── GND

  버튼 미누름 시: GPIO = HIGH (3.3V, 풀업) ✅
  버튼 누름 시:   GPIO = LOW (GND)          ✅
```

### 6.4 PIR 센서 (풀업 불필요)

PIR 센서(HW-MS03, GPIO24)는 센서 자체가 능동적으로 HIGH/LOW를 출력하므로
풀업 저항이 필요하지 않다.

## 7. PIR 센서 (HW-MS03) 전원 변경

### 7.1 현재 문제

회로도에서 PIR 센서(HW-MS03)의 VCC가 **3.3V**에 연결되어 있으나,
제조사(HaiWang) 공식 사양에서 최소 동작 전압은 **3.7V**이다.

| 항목 | 사양 |
|------|------|
| 제조사 공식 VCC 범위 | **3.7V ~ 24V** |
| 현재 회로도 VCC | 3.3V (**사양 미달**) |
| 소비 전류 | ≤6mA |

3.3V로도 동작할 수 있으나, 사양 범위 밖이므로 동작 불안정 및 감지 거리 감소가 발생할 수 있다.

### 7.2 변경 요청

PIR 센서 VCC를 **3.3V → 5V** 레일로 변경한다.

| 항목 | 기존 | 변경 요청 |
|------|------|----------|
| VCC 전원 | 3.3V | **5V** (인터페이스 보드 내부 5V 레일) |
| GND | 변경 없음 | 변경 없음 |
| OUT → GPIO24 | 변경 없음 | 변경 없음 |

### 7.3 OUT 핀 호환성 (변경 불필요 확인)

HW-MS03의 OUT 핀은 VCC 전압과 무관하게 **항상 3.3V HIGH / 0V LOW**를 출력한다.
따라서 VCC를 5V로 변경해도 Jetson GPIO(3.3V 로직)에 **레벨 변환 없이 직결 가능**하며,
OUT → GPIO24 연결은 변경할 필요가 없다.

```
[변경 후]
5V 레일 ──→ HW-MS03 VCC     (3.3V에서 5V로 변경)
GND     ──→ HW-MS03 GND     (변경 없음)
             HW-MS03 OUT (3.3V) ──→ GPIO24 (Pin 18) 직결 (변경 없음)
```

## 8. RS232C 레벨 컨버터 수리

### 8.1 현재 문제

호스트PC와 시리얼 통신 테스트 결과, 인터페이스 보드 내장 RS232C 레벨 컨버터(MAX3232 계열)가 **정상 동작하지 않는다.**

| 측정 항목 | 정상 값 | 실측 값 | 판정 |
|-----------|---------|---------|------|
| RS232 TXD idle 전압 | -8V ~ -12V | **-1.8V** | **NG** |
| 전송 중 전압 변동 | ±8V ~ ±12V 스윙 | **변동 없음 (-1.8V 고정)** | **NG** |

### 8.2 테스트 환경 및 경과

```
[테스트 경로]
Jetson Orin Nano (ttyTHS1, UART1, Pin8 TXD/Pin10 RXD)
  → 40핀 헤더 → 인터페이스 보드 RS232C 컨버터
  → RS232 USB 어댑터 (RS232 레벨) → 호스트PC

[외부 전원] 12V 5A DC 연결 확인
```

- Jetson UART1 pinmux 설정 정상 확인 (TX: tristate=0, RX: enable-input=1)
- Jetson 소프트웨어에서 정상 전송 확인 (pyserial write 성공)
- RS232C 커넥터의 TX/RX 배선 교정 후, 호스트PC에서 **깨진 데이터 수신**
- 양쪽 동일 baudrate (9600~115200 모든 조합)에서도 데이터 깨짐
- RS232 TXD 출력 전압 측정: **-1.8V 고정, 전송 중 변동 없음**
- **차지펌프 미동작으로 RS232 레벨 변환 불가 상태**

### 8.3 추정 원인

1. 차지펌프 캐패시터 (C1+/C1-/C2+/C2-) 미실장 또는 불량
2. RS232 컨버터 칩 자체 불량
3. 칩 VCC (3.3V) 공급 라인 단선

### 8.4 변경 요청

RS232C 레벨 컨버터 회로의 정상 동작을 복구한다.

| 항목 | 내용 |
|------|------|
| 컨버터 칩 확인 | MAX3232 계열 칩 실장 상태 및 동작 확인 |
| 차지펌프 캐패시터 | 4개 (0.1µF~1µF) 실장 상태 확인, 불량 시 교체 |
| VCC 전원 | 칩 VCC 핀에 3.3V 정상 공급 확인 |
| TX/RX 배선 | RS232 커넥터 핀 배치가 DTE/DCE 규격에 맞는지 확인 |
| 출력 전압 기준 | idle 시 TXD = -8V ~ -12V, 전송 시 ±8V ~ ±12V 스윙 |

### 8.5 참고: TX/RX 배선 교정

테스트 중 RS232C 커넥터 ↔ USB 어댑터 간 TX/RX 교차 연결(null modem) 시에만
데이터가 흐르는 것을 확인하였다. 현재 커넥터 핀 배치가 DTE 기준인지 DCE 기준인지
확인 후, 대상 장비(카드 단말기, 호스트PC)와 맞도록 핀 배치를 명시해야 한다.

## 9. 팬 구동 트랜지스터 교체

### 9.1 현재 문제

FAN1, FAN2 구동 회로의 트랜지스터(BJT, BST-CONVERTER/BJT18)가 팬 부하 전류를 감당하지 못한다.

| 측정 항목 | 정상 값 | 실측 값 | 판정 |
|-----------|---------|---------|------|
| FAN 커넥터 전압 (무부하) | 12V | **12V** | OK |
| FAN 커넥터 전압 (팬 연결 시) | 11V 이상 | **2.9~3.9V** | **NG** |
| 트랜지스터 전압 드롭 (Vce) | 0.2~0.5V | **8~9V** | **NG** |

### 9.2 원인 분석

현재 회로 구성:

```
+3.3V ─── R(풀업) ──┬── BJT Base
                     │
GPIO (3.3V) ─── R ──┘

VIN(12V) ──→ BJT ──→ FAN 커넥터 (+)
                      FAN 커넥터 (-) ──→ GND
```

- BJT 베이스 구동 전압이 **3.3V**로, 12V/수백mA 부하를 스위칭하기에 **베이스 전류 부족**
- BJT가 포화(saturation) 상태에 진입하지 못하고 **선형(linear) 영역에서 동작**
- 결과: 트랜지스터에서 **8~9V가 드롭**되어 팬에 2.9~3.9V만 전달 → 팬 구동 불가

### 9.3 검증 결과

| 테스트 | 결과 |
|--------|------|
| 팬에 12V 직접 인가 | **정상 동작** (팬 자체 이상 없음) |
| 인터페이스 보드 FAN 커넥터 무부하 전압 | **12V** (전원 경로 정상) |
| 인터페이스 보드에 팬 연결 시 전압 | **2.9~3.9V** (전류 부족으로 전압 강하) |
| GPIO 소프트웨어 제어 | **정상** (Jetson.GPIO HIGH/LOW 동작 확인) |

### 9.4 변경 요청

FAN1, FAN2 구동 트랜지스터를 **로직 레벨 N-ch MOSFET**으로 교체한다.

| 항목 | 기존 | 변경 요청 |
|------|------|----------|
| 스위칭 소자 | BJT (BST-CONVERTER/BJT18) | **로직 레벨 N-ch MOSFET** |
| 게이트 구동 전압 | 3.3V | 3.3V (변경 없음) |
| 게이트 문턱 전압 (Vgs_th) | - | **1.0~2.0V 이하** (3.3V로 완전 ON) |
| 드레인-소스 저항 (Rds_on) | - | **수십mΩ 이하** |
| 최대 전류 | 부족 | **1A 이상** |

#### 권장 MOSFET 후보

| 부품 | Vgs_th | Rds_on (@Vgs=3.3V) | Id_max | 패키지 |
|------|--------|---------------------|--------|--------|
| AO3400 | 1.0~1.7V | 40mΩ | 5.8A | SOT-23 |
| SI2302 | 0.7~1.3V | 60mΩ | 2.6A | SOT-23 |
| IRLML6344 | 0.8~1.3V | 29mΩ | 5.0A | SOT-23 |

> 핵심: Vgs(th)가 **2.0V 이하**인 로직 레벨 MOSFET을 사용해야 3.3V GPIO로 완전 ON이 보장된다.
> 일반 MOSFET(Vgs_th 3~4V)은 3.3V 구동 시 동일한 문제가 재발한다.

### 9.5 변경 후 회로

```
[변경 후]
+3.3V ─── R(풀업) ──┬── MOSFET Gate
                     │
GPIO (3.3V) ─── R ──┘

VIN(12V) ──→ FAN 커넥터 (+)    ← VIN 직결 (High-side 변경 없음)
              FAN 커넥터 (-) ──→ MOSFET Drain
                                  MOSFET Source ──→ GND
```

또는 기존 High-side 구성을 유지하되, MOSFET 교체만으로 해결 가능:

```
GPIO HIGH (3.3V) → Vgs = 3.3V → MOSFET 완전 ON → Rds ≈ 40mΩ
→ 전압 드롭 ≈ 0.01V (0.3A × 40mΩ) → FAN에 ~12V 공급 → 정상 구동
```

### 9.6 FAN1, FAN2 동일 적용

두 팬 회로 모두 동일한 구조이므로, **FAN1과 FAN2 모두** 트랜지스터를 교체한다.

| 대상 | GPIO | 40핀 헤더 핀 | 교체 대상 |
|------|------|------------|----------|
| FAN1 | GPIO06 (BCM 6) | Pin 31 | BJT → 로직 레벨 N-ch MOSFET |
| FAN2 | GPIO12 (BCM 12) | Pin 32 | BJT → 로직 레벨 N-ch MOSFET |
